# Session Log: 2026-02-28

Last updated: 2026-02-28

## Goal

Build a working submission for the AdderBoard challenge: smallest transformer
that adds two 10-digit numbers with >= 99% accuracy on 10K held-out pairs.

## What Happened

### Track 1: Hand-Coded Model -- SUCCESS

Built a 2-layer decoder with analytically set weights. 100% accuracy on all
10,010 test cases. 249 total parameters.

**Architecture**: 2L decoder, d_model=5, 2 heads (d_head=2), ff=4 per layer.

**Key design decisions**:

1. **Input format**: Reversed digits (LSB-first) with separator tokens.
   `a0 a1 ... a9 SEP b0 b1 ... b9 SEP` (22 tokens).

2. **Fixed attention routing** (0-param buffers): Head 0 gathers A_i + B_i,
   Head 1 gathers A_{i-1} + B_{i-1}. No Q/K projections needed.

3. **Carry detection via clamped ReLU step**:
   diff = (A_{i-1}+B_{i-1}) - S_{i-1} is always in {-1, 0, 9, 10}.
   carry = ReLU(diff - 0.5) - ReLU(diff - 1.5) gives exact {0, 1}.

4. **Mod-10 via two-hinge ReLU**:
   step(S>=10) = ReLU(2S-19) - ReLU(2S-20) is exact for integer S.
   Result = S - 10*step.

5. **Parabolic LM head**: logit[c] = -(x - c - 0.5)^2 peaks at correct digit.

**Critical insight proved during development**: A linear LM head CANNOT perform
modular-10 decode for inputs [0, 19]. Proof: for x=5 and x=15 (both target
digit 5), the linear constraints create contradictions. This means mod-10 MUST
be computed in the MLP, requiring 2 layers to chain carry -> mod10.

### Track 2: Trained Model -- FAILED (2.5% max accuracy)

Architecture: 1L decoder, d_model=7, 1 head, ff=14 (777 params).
Curriculum training (3 phases) + extended grokking phase.
107K steps total, loss dropped from 1.92 to 0.38 but accuracy never exceeded
2.5%. The model learned token-level patterns but couldn't generalize the carry
chain.

**Why it failed**:
- Missing rank-3 factorization (all successful trained models use this)
- No weight tying between embedding and LM head (should reduce param search space)
- Likely needs much longer training or different optimizer settings
- The successful 311-param trained model uses grokking + specific factorization

## Next Steps (Optimization)

The 249-param model has massive room for compression:
- Most weight matrices are >90% zeros
- Only ~30 non-zero weight values actually matter
- Could reduce to ~50-100 params with sparse representations, weight tying,
  rank-1 factorization
- Could potentially match the 36-param leader with ALiBi-based routing instead
  of fixed attention bias matrices

## Reflections

- Starting with hand-coded was the right call. Training transformers for
  arithmetic from scratch requires very specific tricks (rank factorization,
  specific data formats, long grokking phases) that are hard to get right.
- The mathematical analysis of carry detection (diff in {-1,0,9,10}) was the
  key insight that made the hand-coded model tractable.
- Proving that linear LM heads can't do mod-10 saved time -- it immediately
  ruled out 1-layer architectures and pointed to the 2-layer solution.
